Shop & Restaurant Dynamic Pricing System

1 • Problem Statement & Solution Overview

Problem Statement:

  • Demand Fluctuation.
  • High Wastage in Restaurants.
  • Unpredictable Customer Traffic.
  • Inventory Mismanagement.
  • Competition Pressure.

Solution Overview:

  • Monitors demand and stock.
  • Adjusts prices dynamically using a formula.
  • Prioritizes items for discounts using a heap.
  • Routes supplier orders using Dijkstra.
  • Keeps ordered data in balanced trees for quick range queries and updates.

2 • Algorithms & Data Structures

Algorithm / Data Structurepurpose
AVL Treefor ordered item storage (by price or id).
BSTfor simple ordered storage/example.
Stack & Queuefor order processing pipeline
Heaps (min/max)for top-demand and urgent-discount items
Hashing (unordered_map)for fast id → item lookup
Quick/Merge/Heap Sortfor reports and analytics.
Binary Searchon sorted price lists.
Graph + Dijkstrafor finding nearest supplier / fastest replenishment route
