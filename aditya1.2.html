Smart City — Hospital & Emergency Resource Management

1 • Problem Statement & Solution Overview

Problem Statement:

  • Ambulances not allocated optimally leading to delays.
  • ICU beds get filled; no real-time availability updates.
  • Delay in assigning doctors and nurses.
  • Blood banks unable to track urgent blood requirements.
  • Equipment shortage (ventilators, oxygen cylinders).
  • Emergency triage not prioritised properly.

Solution Overview:

  • Ambulance Routing & Traffic Navigation (Dijkstra + dynamic updates)
  • Patient Triage & Prioritization (Max-Heap / Priority Queue)
  • Bed / ICU / Ventilator Allocation (AVL / BST)
  • Medicine Stock & Distribution (Hash table + Min-Heap)
  • Emergency Prediction (Dynamic Programming / Greedy)

3 • System Architecture

  • Ambulance Routing Module

    Graph of city roads. Use Dijkstra to compute shortest path (with dynamic edge weights for traffic).

  • Patient Priority Module

    Max-Heap based triage queue to pick highest severity first.

  • Resource Allocation Module

    AVL Tree for beds / ICUs; Hash table for medicines with fast lookup.

  • Waste Management

    Min-Heap ordered by expiry date to reduce medicine waste.

  • Emergency Prediction Module

    DP & Greedy models to forecast demand and pre-allocate resources.

4 • Time & Space Complexity Analysis

  • Dijkstra’s Algorithm — Time: O(E log V), Space: O(V)
  • Priority Queue (Patient Priority) — Push / Pop: O(log n)
  • AVL Tree (Bed Allocation) — Insert / Search / Delete: O(log n), Space: O(n)
  • Hash Table (Medicine Stock) — Average O(1) lookup
  • Min-Heap (Expiry) — Insert / min() / pop: O(log n)
