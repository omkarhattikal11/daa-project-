Business Case 8: Emergency Evacuation Path Planner

1. Business Case Description

During emergencies such as fires, building accidents, or campus threats, people often panic and choose unsafe or crowded routes.

The Emergency Evacuation Path Planner identifies the safest and fastest exit paths inside buildings, malls, schools, and public spaces. By dynamically analyzing routes, it guides people to safety efficiently, avoiding confusion or delay.

2. Problem Statement

  • Blocked corridors
  • Hazards or fire spread
  • Crowd movement or congestion

System Requirements

  • Analyze all possible exits in real time
  • Detect obstacles immediately
  • Compute the safest and fastest escape route instantly

3. Solution Approach

  • Graph represents building layout (rooms, corridors, exits)
  • BFS finds shortest paths in equal-cost areas
  • Dijkstra selects safest routes in weighted-risk areas
  • Queue supports BFS traversal for orderly node exploration

System Output

  • Safest and fastest evacuation paths
  • Step-by-step guidance for users
  • Real-time updates for blocked or hazardous paths

4. Algorithms and Data Structures Used

BFS (Breadth-First Search) — Fastest Path in Equal-Weight Areas

Finds the shortest unweighted path to the nearest exit.
Works efficiently in grid-like layouts such as classrooms or hallways.

Dijkstra’s Algorithm — Weighted Escape Route Selection

Calculates the safest and fastest route considering path risks or delays.
Accounts for crowd density, hazards, or staircases.

Graph (Adjacency List) — Representing Building Layout

Models rooms, corridors, and exits as nodes and edges.
Allows dynamic updates and efficient traversal during emergencies.

Queue — Node Processing for BFS

Processes nodes level by level in BFS traversal.
Ensures the system finds the quickest path efficiently.

5. System Workflow

Start
↓
Store Building Layout in Graph
↓
Update Obstacles in Real Time
↓
Run BFS for Equal-Cost Paths
↓
Run Dijkstra for Weighted-Risk Paths
↓
Highlight Safest and Fastest Evacuation Path
↓
Provide Step-by-Step Guidance to Users
↓
End

6. Space and Time Efficiency

Time Complexity

  • BFS: O(V + E), V = nodes, E = edges
  • Dijkstra using Min-Heap: O((V + E) log V)
  • Queue operations for BFS: O(1) per node

Space Complexity

  • Graph adjacency list: O(V + E)
  • Distance and visited arrays: O(V)
  • Queue storage for BFS: O(V)

7. Sustainable Development Goals (SDGs) Alignment

  • SDG 3: Good Health and Well-Being

8. Conclusion

The Emergency Evacuation Path Planner provides an intelligent, real-time solution for emergency safety. By combining BFS, Dijkstra, and graph-based modeling, it identifies the safest and fastest evacuation routes, helping people reach safety efficiently during emergencies.
